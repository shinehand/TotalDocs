# HWP Spec Analysis

이 디렉터리는 HWP 형식 문서를 구현 관점에서 다시 정리한 분석 Markdown을 모은다.

## 목적

- 파서 구현자가 레코드/속성/값 범위를 빠르게 찾을 수 있게 한다.
- 렌더러 구현자가 레이아웃에 직접 영향을 주는 필드를 놓치지 않게 한다.
- 저장/복원/편집 경로가 어떤 필드를 반드시 유지해야 하는지 한눈에 보이게 한다.

## 최종 심화 검토 상태

- 2026-04-18 기준으로 형식 PDF 5종 전체를 다시 끝까지 검토했다.
- 텍스트 추출만으로 불명확한 표, 도식, 예제는 필요한 경우 PDF 렌더링 이미지까지 다시 확인했다.
- 현재 이 디렉터리의 개별 분석 문서는 모두 `구현 입력용 매뉴얼` 수준을 목표로 보강되었다.
- 이후 개발 단계에서는 이 문서들을 요약본으로 다시 줄이지 말고, 필요한 장/절을 직접 재참조하는 방식으로 쓰는 편이 안전하다.

## 문서 작성 규칙

- 각 PDF마다 Markdown 1개를 만든다.
- 단순 요약이 아니라 `정확한 데이터 처리`와 `레이아웃 처리` 기준이 되도록 적는다.
- 가능하면 다음 항목을 모두 포함한다.

## 공통 섹션

1. 문서 개요
2. 형식 범위와 버전
3. 핵심 데이터 구조
4. 파서가 반드시 뽑아야 할 필드
5. 레이아웃에 직접 영향을 주는 필드
6. 편집/저장 시 보존해야 할 필드
7. 구현 체크리스트
8. 즉시 우선순위
9. 원문에서 다시 봐야 할 장/절
